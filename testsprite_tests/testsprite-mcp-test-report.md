## 4️⃣ Key Gaps / Risks
- The UI currently lacks a direct “go to date” or task search mechanism, making far-future test scenarios (e.g., 2030) impractical and causing several false-negative CRUD failures.
- Some edit/delete validations depend on reliably reopening items after creation; this path showed flaky behavior and should be stabilized with clearer selectors and deterministic task targeting.
- Running in dev mode can increase flakiness under automated E2E load; production preview mode is recommended for higher signal quality.
- Until timeline navigation/search is improved, E2E coverage for full lifecycle scenarios on arbitrary dates remains limited and may underreport real functionality quality.
